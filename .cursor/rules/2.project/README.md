# 📱 프로젝트 규칙 (Project Rules)

이 프로젝트에서 사용하는 도메인 용어, 언어별 코딩 규칙, UI/UX 가이드라인입니다.

## 📋 파일 목록

### 1. [development-dictionary.mdc](./development-dictionary.mdc) 🔴 필수
**개발 용어 사전**
- 프로젝트 전체에서 사용하는 도메인 용어 정의
- Branch, PlaceGroup, Place 등 핵심 개념
- 용어 일관성 유지를 위한 규칙
- Ubiquitous Language (보편 언어)

### 2. [flutter-rule.mdc](./flutter-rule.mdc) 🔴 필수
**Flutter 코딩 규칙**
- Flutter 및 Dart 코딩 스타일
- MVVM 패턴 구현 방법
- Riverpod 상태 관리
- 위젯 구성 및 최적화

### 3. [ui-ux-guidelines.mdc](./ui-ux-guidelines.mdc) 🟡 중요
**UI/UX 가이드라인**
- 디자인 시스템 (색상, 타이포그래피, 간격)
- 반응형 디자인 (브레이크포인트, 레이아웃)
- 공통 UI 컴포넌트
- 애니메이션 및 트랜지션
- 접근성 (Accessibility)

## 🎯 사용 시기

- **프로젝트 시작 전**: development-dictionary.mdc로 도메인 용어 학습
- **코드 작성 시**: flutter-rule.mdc 준수
- **UI 구현 시**: ui-ux-guidelines.mdc 참고
